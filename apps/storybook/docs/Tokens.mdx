# Design Tokens

Todos os tokens usam o formato **W3C Design Tokens (DTCG)** com `$value` e `$type`,
compilados pelo **Style Dictionary v4** para CSS Custom Properties.

## Primitivos (Base)

Valores brutos, iguais em todas as marcas.

## Component Tokens

Tokens específicos de cada componente, referenciando primitivos.

### Button
| Token | Valor |
|-------|-------|
| `--ds-component-button-border-radius` | `{radius.md}` → `8px` |
| `--ds-component-button-font-weight` | `{font.weight.medium}` → `500` |
| `--ds-component-button-md-height` | `40px` |
| `--ds-component-button-md-padding-x` | `{spacing.4}` → `16px` |

### Input
| Token | Valor |
|-------|-------|
| `--ds-component-input-border-radius` | `{radius.md}` → `8px` |
| `--ds-component-input-md-height` | `40px` |
| `--ds-component-input-md-padding-x` | `{spacing.4}` → `16px` |

## Uso nos Componentes

Componentes **nunca** usam valores hardcoded. Sempre via CSS custom properties:

```css
/* ✅ Correto */
.button {
  border-radius: var(--ds-component-button-border-radius);
  background: var(--ds-color-action-primary);
}

/* ❌ Errado */
.button {
  border-radius: 8px;
  background: #6366f1;
}
```

## Tokens Studio Sync

Os tokens são compatíveis com **Tokens Studio Git Sync**:

- Formato DTCG (`$value`, `$type`)
- Referências via `{color.indigo.500}`
- Arquivos JSON em `packages/tokens/src/`
